# SFAO Portal Phase Plan

## Goal
Deliver the portal in clear phases, starting with access control and operations flow, then adding social and survey modules.

## Phase 0 - Current Baseline
- Login and dashboard entry
- Basic feedback feed and analytics views

## Phase 1 - Access Foundation
- Role-aware portal menu
- Department shown on user profile/header
- Hide modules user should not access

## Phase 2 - Operations Core
- Department queue view
- Filters: department, urgency, status
- Assign/Re-route controls

## Phase 3 - Social Monitoring UI
- Social post cards in dashboard
- Sentiment badge per post
- Post details popup (analysis, urgency, suggested action)

## Phase 4 - Survey Builder UI
- Template list
- Create/edit/duplicate template
- Publish/unpublish template

## Phase 5 - Resolver Workflow
- Department workboard
- Status flow: New -> In Progress -> Resolved
- Escalation and SLA indicators

## Phase 6 - Buyer Tailoring (Safaricom)
- Safaricom department map
- Safaricom issue categories
- Safaricom KPIs and demo scenario

## Suggested Build Order
1. Phase 1
2. Phase 2
3. Phase 3
4. Phase 4
5. Phase 5
6. Phase 6

## Definition of Done Per Phase
- UI visible and usable
- API integrated (if phase needs backend)
- Basic test/validation pass
- Short demo path documented
